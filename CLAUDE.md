This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
cargo build                  # debug build
cargo build --release        # release build
cargo test --all             # run all tests
cargo clippy --all --all-targets  # lint
cargo fmt                    # format code
cargo fmt -- --check         # check formatting without writing
```

Code coverage (requires `cargo install cargo-llvm-cov`):
```bash
cargo cov          # terminal summary
cargo cov-html     # HTML report (opens in browser)
cargo cov-json     # JSON report
```

To run a single test, use the standard Rust pattern:
```bash
cargo test <test_name>       # e.g. cargo test test_noder_let_stmt
```

Test names are auto-generated by `build.rs` from `.manta` source files in `tests/src/`.

## Architecture

Manta is a WIP compiler for a systems programming language (manual memory management, `new`/`free`, `try`/`catch`, `defer`, pattern matching). The pipeline is:

```
Source → Lexer → Parser → AST → Noder → HIR → Typer → Blocker → MIR → [LLVM IR, TBD]
```

**Stages:**
- **Lexer** (`src/parser/lexer.rs`): Tokenizes source into `Token`/`TokenKind`.
- **Parser** (`src/parser.rs`, `src/parser/`): Recursive descent with parselets for expressions. Produces an `AST` (enums `Decl`, `Expr`, `Stmt`, `Pattern`).
- **Noder** (`src/noder.rs`): Lowers AST → HIR (`NodeTree`). Each construct becomes a `Node` enum variant identified by `NodeID`. Importantly, the Noder performs identifier resolution: every use of a named identifier is assigned the **same `NodeID` as its declaration**. This means `NodeID` serves as the resolved symbol identity — no separate resolution pass is needed in later stages. Code in the Blocker and beyond can treat an identifier's `NodeID` as a direct key into `local_map` or other declaration-keyed tables.
- **Typer** (`src/noder/typer.rs`): Type checks and infers types over the `NodeTree`, populating a `type_map` side table (`NodeID → TypeSpec`).
- **Blocker** (`src/blocker.rs`): Lowers HIR → MIR, building a control-flow graph of `BasicBlock`s with SSA-like `ValueId`s.

**Key data structures:**
- `NodeTree`: Holds all HIR nodes, `type_map`, and `symbol_map` side tables.
- `SideTable<K,V>`: BTreeMap + Vec store used for `type_map` (`NodeID → TypeSpec`) and `symbol_map` (`SymID → NodeID`).
- `TypeSpec`: Type information including `Any` (unknown), `IntLiteral(i64)`/`FloatLiteral(f64)` (unresolved numeric literals), `InferredEnumExpr/Pat` (partial enum info), `Panic` (unreachable), `Unit` (void).
- `StrStore` / `StrID`: String interning — all identifiers are stored once and referenced by integer ID.
- `MirFunction`: Contains `LocalId`-indexed locals, `BlockId`-indexed `BasicBlock`s, and an entry block.

## Testing

Tests are **snapshot tests** generated by `build.rs`. For each `tests/src/*.manta` file, test functions are generated that compare compiler output (JSON) against expected files in `tests/{lexer,parser,noder,blocker}/`. Each stage validates its own output independently.

To add a new test: add a `.manta` file to `tests/src/` and corresponding expected JSON files in the relevant stage directories.
